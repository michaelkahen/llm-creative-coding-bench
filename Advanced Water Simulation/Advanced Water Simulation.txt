Create a single self-contained HTML file that simulates a realistic body of water
with dynamic, randomly-generated waves, rendered top-down or at a raking angle,
using only HTML, CSS, and vanilla JavaScript (canvas 2D context — no WebGL, no

Requirements:

1. WAVE SIMULATION (not decoration)
   - Do NOT simply overlay a few `sin(x + t)` waves and call it done. Implement an

     discretized wave equation (e.g. a simple shallow-water / wave-propagation

2. INTERACTIVITY
   - Clicking or dragging on the water should inject a disturbance at that point

   - Optional foam/whitecap effect: mark cells as "foam" when local height velocity
     or curvature crosses a threshold, and fade the foam out over a few frames.

4. PERFORMANCE
   - Must run smoothly via requestAnimationFrame at a grid resolution that's dense
     enough to look fluid (guideline: at least ~100x100 simulation cells, upscaled
     to fill a larger canvas via interpolation — do not simulate at full pixel
     resolution, that's a performance trap).
   - Explain any resolution/performance tradeoffs you made in the write-up below.

5. CONSTRAINTS
   - One single .html file, valid and runnable by opening directly in a browser.
   - No external dependencies, no network calls, no images.
   - Code must be clean, with comments explaining: the wave propagation update

   - Must run without console errors, and must not degrade into visual noise or
